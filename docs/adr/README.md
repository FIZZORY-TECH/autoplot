# Architecture Decision Records

This directory captures the binding architectural rules of the autoplot codebase in MADR-lite form. Each ADR cites the exact file and line range where the rule is enforced today, so an agent can verify the constraint is real before acting on it. ADRs are append-only: superseding a decision means writing a new ADR that references the old one, never editing history.

1. [ADR-0001 — MarketDataProvider Interface Is Frozen (A3)](./0001-market-data-provider-frozen.md) — the canonical adapter contract (`fetchHistory`, `subscribeRealtime`, `search`) cannot be mutated; new providers conform to it.
2. [ADR-0002 — Timeframe Set Locked to 4-Tier (G-4)](./0002-timeframe-set-locked.md) — `Tf` is `'1h' | '4h' | '1d' | '1w'` and nothing else, regardless of what the prototype displays.
3. [ADR-0003 — Claude CLI Profile Isolation (Wave 0)](./0003-claude-profile-isolation.md) — every `claude` subprocess runs against `<data_dir>/autoplot/claude-home/` with leaky env vars stripped; the user's `~/.claude*` is never touched.
4. [ADR-0004 — Design Adoption Is Binding](./0004-design-binding-prototype.md) — `app-design/project/` is the approved spec; production must achieve token, motion, and component-shape parity, and prototype files are never copied into `src/`.
5. [ADR-0005 — Append-Only SQLite Migrations (A1)](./0005-append-only-migrations.md) — migrations are embedded via `include_str!`; new schema changes append a new file + array entry, prior migrations are never edited.
6. [ADR-0006 — Terminal PTY Mode](./0006-terminal-pty-mode.md) — `portable-pty` 0.9 hosts real interactive PTY sessions; spawns reuse the same `isolation_flags()` + `ENV_REMOVE_KEYS` as `ai_invoke`; base64 framing on `terminal:data` events; xterm lazy-loaded (~250 KB, absent in browser mode).
7. [ADR-0007 — App MCP Bridge](./0007-app-mcp-bridge.md) — `autoplot-mcp` sidecar dials the app's UDS/named-pipe IPC bridge; per-launch rotated token handshake; 38 tools across read-only/mutation/persistence/portfolio/compute buckets; mutation gated by `MCPConsentToast`; all three AI modes gain the full `mcp__autoplot__*` toolset automatically.
8. [ADR-0008 — Equities Provider Added](./0008-equities-provider.md) — supersedes the closed-union portion of ADR-0001: widens `Provider` to include `'alpaca'` and `AssetClass` to include `'equity'`; freezes `list_assets` return shape, `provider`-mandatory-in-WHERE invariant across `bars`/`marks`/`watchlist`/`trends`, UTC-bucketed 4h alignment for equities, and the `var(--emerald)` equity accent token (pending definition in `tokens.css`).
9. [ADR-0009 — Dynamic Symbol Catalog + `(provider, sym, quote)`-Mandatory-In-Key](./0009-dynamic-symbol-catalog.md) — extends ADR-0008's WHERE-clause invariant with a mandatory `quote` dimension; introduces a SQLite-resident catalog with FTS5 search, `_v2` tables for the over-constrained `watchlist` and `bars` PKs, deterministic per-provider quote backfill (Kraken audited per-row), and a `CatalogFetcher` trait kept off the frozen `MarketDataProvider`.
10. [ADR-0010 — Portfolio Holdings Model](./0010-portfolio-holdings-model.md) — denormalized one-row-per-`(sym,provider,quote)` holdings table with weighted-average cost; no transaction ledger or FIFO in v1; value/P&L computed at read time; agent mutations gated by MCP consent.
11. [ADR-0011 — Activity-Bar + Drawer Dock](./0011-activity-bar-drawer-dock.md) — **amends ADR-0004** and supersedes the floating-panel layout: two ~48px activity rails toggle one-per-side `DockDrawer`s that inset the chart via `--rail-w` + the reserve vars; `useDockStore` is the single source of drawer open-state; Terminal open by default; `mountOnOpen` teardown policy for PTY/CodeMirror/Settings; the panel-UI-only Overlays→Indicator rename; ≥240px narrow-window clamp. Keeps ADR-0004's binding-fidelity principle (the prototype now depicts the dock).
