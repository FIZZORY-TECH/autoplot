# Glossary

Domain terms used throughout this codebase. Definitions are deliberately one-sentence; cite source paths so an agent can verify before acting.

- **ai_invoke** — ~~Tauri command channel that fronted the Claude CLI subprocess~~ **REMOVED 2026-05-23.** The CLI is now accessed exclusively via the Terminal PTY (`terminal_spawn`). Source: historical ref in `src-tauri/src/commands/` git log.
- **Alpaca** — Equities market data provider used for stocks. Free IEX feed (30 WS symbols, 200 REST req/min) or $9/mo SIP. See `src-tauri/src/providers/alpaca.rs` and `src/data/adapters/alpaca.ts`.
- **App sidecar** — The standalone `autoplot-mcp` binary registered in `tauri.conf.json` `bundle.externalBin`; shipped with the app, launched by the Claude CLI via `mcp.json`, and exposes app capabilities over MCP/stdio. Source: `src-tauri/sidecars/autoplot-mcp/`.
- **AssetMeta** — Canonical descriptor for a tradable asset (token, display name, provider hints) returned by providers and rendered in the asset panel. Source: `src/data/MarketDataProvider.ts`.
- **audit log** — ~~Opt-in newline-delimited JSON trace of AI tool calls~~ **REMOVED 2026-05-23.** Historical: was generated by the removed chat UI. Source: historical ref at `src/ai/audit.ts` git log.
- **backtest** — Deterministic pass of a Strategy over a Dataset that emits signals plus summary stats; runs entirely in-process, never against a real broker. Source: `src/ai/tools/`.
- **Bar** — One OHLCV row at a given Tf, the atomic unit consumed by the chart pipeline and overlay calculators. Source: `src/data/MarketDataProvider.ts`.
- **buildOverlays** — Function that assembles the active overlay-renderer list from overlay flags, user custom series, and the always-on rangeScope overlay. Source: `src/chart/overlays.ts`.
- **canonical token** — Provider-agnostic asset identifier (e.g. `BTC`) used everywhere in app state; adapters map it to provider-specific symbols (`BTCUSDT`, `BTC-USD`, `XBT/USD`) at the wire boundary. Source: `src/data/adapters/`.
- **ChartCanvas** — The single Canvas2D React component that owns the chart pipeline, driven by `bars`, `view`, a `ChartRenderer`, and overlay renderers. Source: `src/chart/ChartCanvas.tsx`.
- **ChartRenderer** — One of six pluggable renderers (candles, line, area, bars, baseline, hollow-candles) selected to morph the chart-type without rebuilding state. Source: `src/chart/renderers/`.
- **claude-home** — App-managed Claude CLI profile directory created under `<data_dir>/autoplot/` so the app's CLI state is isolated from the user's `~/.claude`. Source: `src-tauri/src/profile.rs` (profile isolation enforced for both deleted chat UI and surviving Terminal mode).
- **comments** — Persisted free-text annotations attached to a chart coordinate, rendered alongside marks and trends. Source: `src/chart/marks.ts`.
- **Dataset** — Zod-validated descriptor of a time-bounded slice of bars (asset, Tf, range, optional indicators) that strategies and backtests consume. Source: `src/ai/schemas.ts`, `docs/schemas/dataset.schema.json`.
- **Equity** — Asset class for stocks/ETFs (AAPL, MSFT, SPY, QQQ etc). Distinguished from `crypto` in `AssetClass` per ADR-0008.
- **glow overlay** — Soft-luminance pass drawn beneath line/area renderers to give the cinematic dark-glass feel; one of the standard overlay layers. Source: `src/chart/overlays.ts`.
- **Holding** — a position the user owns: qty + weighted-average cost for a `(sym, provider, quote)`, stored in `portfolio_holdings`; value/P&L computed at read time. Source: `src-tauri/migrations/0018_portfolio.sql`.
- **hydrate** — Startup routine that loads persisted Zustand slices from SQLite and wires the debounced write-back subscription. Source: `src/lib/hydrate.ts`.
- **hydrated flag** — Boolean from `hydrate.ts` that gates write-back so defaults aren't accidentally persisted before load completes. Source: `src/lib/hydrate.ts`.
- **IEX feed** — Free Alpaca data tier sourcing trades from IEX exchange only. Used as default; SIP consolidated feed is the paid alternative.
- **indicator skill** — CLI skill file (`src-tauri/resources/profile-assets/skills/pinescript-to-indicator/SKILL.md`) that teaches the in-app Claude CLI agent to convert a pasted **Pine Script** or a natural-language request into a **ResearchOverlay** by orchestrating `compute_indicator` + `apply_research_overlay`. Always emits a top-level `recipe` field; applies via a single MCP consent (persist-on-apply is automatic). Produces a static snapshot (≤500 points, no tick updates). See [ADR-0013](adr/0013-pinescript-indicator-overlay-recipe.md), [ADR-0014](adr/0014-overlay-recipe-recompute-model.md). Source: `src-tauri/resources/profile-assets/skills/pinescript-to-indicator/SKILL.md`.
- **Indicator enum** — Closed 15-entry enum of indicator kinds an AI agent may attach to a Dataset (SMA/EMA/RSI/MACD/etc.). Source: `src/ai/schemas.ts`, `docs/schemas/dataset.schema.json`.
- **marks** — Persisted point/box annotations stored in SQLite and drawn by the chart marks layer. Source: `src/chart/marks.ts`.
- **MCP bridge** — The local Unix-domain-socket / named-pipe IPC server in `ipc_bridge.rs` that the `autoplot-mcp` sidecar dials to expose app capabilities (chart state, persistence, compute) to the Claude CLI. Source: `src-tauri/src/ipc_bridge.rs`, [ADR-0007](adr/0007-app-mcp-bridge.md).
- **MarketDataProvider** — FROZEN interface (Architectural Decision A3) defining `assets`, `bars`, `realtime`, plus the locked 4-tier `Tf` set; never mutate. Source: `src/data/MarketDataProvider.ts`.
- **MCP merged** — Concept that the app merges its own MCP server config with the user's existing Claude config so user-installed tools remain available alongside app-provided ones. Source: `src-tauri/src/commands/mcp.rs`.
- **MockMarketDataProvider** — Deterministic in-memory provider used in browser-only dev, Playwright, and as the registry fallback when Tauri runtime or a real adapter is missing. Source: `src/data/mockProvider.ts`.
- **ohlcCache** — In-memory LRU keyed by (provider, token, Tf, range) that absorbs repeated REST history requests during pan/zoom. Source: `src/data/ohlcCache.ts`.
- **Op enum** — Closed 8-op enum of comparison/logical operators a Strategy Condition may use. Source: `src/ai/schemas.ts`, `docs/schemas/strategy.schema.json`.
- **oscillator sub-pane** — A separate chart pane rendered below the price pane, on an independent y-scale, used for bounded or unbounded oscillators (e.g. RSI 0–100, ATR). Activated by setting `pane: 'series'` on a `LineElement`, `BandElement`, or `HLineElement` in a **ResearchOverlay**; only one oscillator sub-pane is supported at a time. See [ADR-0013](adr/0013-pinescript-indicator-overlay-recipe.md). Source: `src/ai/schemas.ts`.
- **overlay** — A renderer drawn on top of (or beneath) the base chart series — indicators, range-scope, glow, signals, marks. Source: `src/chart/overlays.ts`.
- **permission mode** — Claude CLI flag controlling whether tool calls auto-approve, prompt, or are blocked; surfaced in Terminal settings. Source: `src-tauri/src/profile.rs`, `src/terminal/`.
- **Pine Script** — TradingView's proprietary indicator scripting language. The **indicator skill** converts a subset of Pine Script constructs (SMA, EMA, RSI, ATR, Bollinger, Donchian, `hline`) into a **ResearchOverlay** via `compute_indicator`; unsupported constructs (`ta.macd`, `ta.stoch`, `plotshape`, `security()`) are rejected with a plain-language explanation. Source: `src-tauri/resources/profile-assets/skills/pinescript-to-indicator/SKILL.md`.
- **Portfolio** — the set of all holdings; surfaced in the Portfolio panel and via the `portfolio_*` MCP tools for agent analysis. Source: `src/panels/PortfolioPanel.tsx`.
  
- **PTY session** — An interactive pseudo-terminal hosted by `src-tauri/src/commands/terminal.rs`. One session is created per `openTerminal` call; capped at 4 concurrent sessions. Reader thread emits `terminal:data` and `terminal:exit` Tauri events; writes arrive via `terminal_write`. Source: `src-tauri/src/commands/terminal.rs`, [ADR-0006](adr/0006-terminal-pty-mode.md).
- **PFIX** — The post-P4 fix/audit wave that added the Trend Line tool and a prototype-fidelity style audit. Source: `docs/audit/PFIX-style-audit.md`.
- **Plan mode** — Claude CLI mode in which the agent drafts a plan before executing tools. Source: `src/terminal/` (Terminal mode only).
- **Provider (type)** — String-literal union of supported providers (`'mock' | 'binance' | 'coinbase' | 'kraken'`) used to route registry lookups. Source: `src/data/MarketDataProvider.ts`.
- **providerRegistry** — Lookup that returns a `MarketDataProvider` for a given provider id, with `Mock` as the deterministic fallback. Source: `src/data/providerRegistry.ts`.
- **rangeScope** — Mini-map style overlay showing the full series with a draggable window indicating the current `ViewWindow`; uses a ref to avoid morph re-trigger. Source: `src/chart/rangeScope.ts`.
- **realtime** — Streaming layer that subscribes to provider WebSockets in TS and emits incremental bar updates into the chart and watchlist. Source: `src/data/realtime.ts`.
- **recipe** — Machine-readable indicator spec (`RecipeSpec`) embedded as an optional top-level field on a `ResearchOverlay`. Stores the logical series to compute (kind, params, pane, color) separately from the rendered value arrays, enabling the overlay to be recomputed against different bars via `recomputeRecipe`. A recipe-bearing overlay is auto-saved to the Research Library on `apply_research_overlay` (one-consent model). See [ADR-0014](adr/0014-overlay-recipe-recompute-model.md). Source: `src/ai/schemas.ts`.
- **recomputeRecipe** — Pure frontend function that re-runs a saved overlay's `recipe` against caller-supplied bars, producing a fresh `ResearchOverlay` for the current `(sym, tf)`. Mirrors `computeIndicator.ts`'s per-kind dispatch exactly. No React, no store reads, no I/O. See [ADR-0014](adr/0014-overlay-recipe-recompute-model.md). Source: `src/panels/recomputeRecipe.ts`.
- **RecipeSpec** — Zod sub-schema `{ source: 'pine' | 'nl', series: SeriesSpec[] }` carried by `ResearchOverlay.recipe`. `source` mirrors the top-level overlay provenance; `series` is the ordered list of indicator series to compute on reapply. See [ADR-0014](adr/0014-overlay-recipe-recompute-model.md). Source: `src/ai/schemas.ts`.
- **Research Library** — Right-rail drawer listing saved `research_overlays` and datasets with re-apply / switch-and-re-apply / delete actions; backed by `useResearchOverlayLibraryStore` and the `save/list/load/delete_research_overlay` MCP tools. Source: `src/panels/ResearchLibrary.tsx`, `src/stores/useResearchOverlayLibraryStore.ts`.
- **Research mode** — ~~One of the three AI modes~~ **REMOVED 2026-05-23.** Historical: was a chat-UI feature. Source: historical ref in git log.
- **research_overlays (table)** — SQLite blob table (migration `0019_research_overlays.sql`) persisting `ResearchOverlay` JSON for the Research Library; mirrors the `datasets` table pattern. Columns: `id TEXT PK`, `name TEXT`, `sym TEXT`, `tf TEXT`, `blob TEXT`, `created_at INTEGER`. Source: `src-tauri/migrations/0019_research_overlays.sql`, `src-tauri/src/commands/db.rs`.
- **ResearchOverlay** — Zod-validated rich chart overlay bundling up to 50 heterogeneous `Element`s (line, band, hline, markers, event_mark, text, hotspot) keyed to one `(sym, tf)`; applied via the `apply_research_overlay` MCP tool and held in `useChartMutationStore.researchOverlays`. Saved to the `research_overlays` table and managed via the Research Library. Preferred over a `Dataset` for anything beyond a single bare numeric series. **Additive fields (backward-compatible):** `source?: 'pine' | 'nl'` — drives a provenance badge in LegendHUD; absent means no badge. `recipe?: RecipeSpec` — machine-readable indicator spec; when present, `apply_research_overlay` auto-saves to the library (one-consent model) and `recomputeRecipe` can re-run it against new bars (see **recipe**, [ADR-0014](adr/0014-overlay-recipe-recompute-model.md)). `LineElement`, `BandElement`, and `HLineElement` each carry `pane?: 'price' | 'series'`; absent defaults to `'price'` (see **oscillator sub-pane**). Source: `src/ai/schemas.ts`, `src/stores/useChartMutationStore.ts`.
- **SeriesSpec** — Zod sub-schema `{ kind, params?, pane?, color?, width? }` describing one series entry inside a `RecipeSpec`. `kind` is the `Indicator` enum extended with the logical aliases `'bollinger'` and `'donchian'` (recipe-only; not added to the main enum). Source: `src/ai/schemas.ts`.
- **signals** — Backtest output drawn on the chart as triangles plus connector lines between entry and exit. Source: `src/chart/signals.ts`.
- **SIP** — Securities Information Processor; the consolidated US-equities tape covering all exchanges. Paid Alpaca tier.
- **snake_case wire format** — Convention that Rust structs serialize with snake_case fields and TS mirrors them exactly so no per-row remapping is needed at the IPC boundary. Source: `CLAUDE.md`.
- **stream-json** — Claude CLI output format. ~~The app parsed this in the removed chat UI~~ Now used only by Terminal mode. Source: `src/terminal/`.
- **Strategy** — Zod-validated descriptor of an entry/exit ruleset built from Conditions over indicators, runnable against a Dataset. Source: `src/ai/schemas.ts`, `docs/schemas/strategy.schema.json`.
- **Strategy Condition** — Single comparison node (`indicator op value` or `indicator op indicator`) — the leaf type composed into a Strategy's logic tree. Source: `src/ai/schemas.ts`, `docs/schemas/strategy.schema.json`.
- **Strategy mode** — ~~One of the three AI modes~~ **REMOVED 2026-05-23.** Historical: was a chat-UI feature. Source: historical ref in git log.
- **Terminal mode** — The third AI surface, peer of Research and Strategy. Runs the real interactive `claude` CLI TUI inside `XtermPanel.tsx` via a PTY session, with hard-locked app-profile isolation identical to `ai_invoke`. Source: `src/terminal/`, [ADR-0006](adr/0006-terminal-pty-mode.md).
- **Tf** — FROZEN 4-tier timeframe set `'1h' | '4h' | '1d' | '1w'` (per G-4); 5m/15m are explicitly excluded even though the prototype shows them. Source: `src/data/MarketDataProvider.ts`.
- **trends** — Persisted line-segment annotations (trend lines added in PFIX) drawn by the trends layer. Source: `src/chart/trends.ts`.
- **ViewWindow** — `{ start, end, yMin, yMax }` describing the visible bar slice and y-bounds; recomputed by `AppShell` on every pan/zoom. Source: `src/chart/types.ts`.
- **Wave 0** — Post-P5 polish wave covering profile isolation (`claude-home`), AI chat UX, and first-run/auth fixes. Source: `docs/plan/HAND-OFF.md`.
